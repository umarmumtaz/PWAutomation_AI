# GitHub & Jenkins Integration Guide

## Overview

This project is configured for **continuous integration** with both:
- **GitHub Actions** – automatic CI/CD on push/PR
- **Jenkins** – additional CI/CD pipeline for enterprise deployments

---

## GitHub Actions Setup

### Current Status ✅
Your GitHub Actions workflow is **already configured** at `.github/workflows/playwright.yml`

### How It Works

1. **Triggers** on:
   - Push to `main` or `master` branches
   - Pull requests to `main` or `master` branches

2. **What It Does**:
   - Checks out code
   - Installs Node.js and dependencies (`npm ci`)
   - Installs Playwright browsers & system deps (`npx playwright install --with-deps`)
   - Runs all tests (`npx playwright test`)
   - Uploads HTML test report as artifact (30-day retention)

3. **View Results**:
   - Go to your GitHub repository
   - Click **Actions** tab
   - Select the workflow run
   - Scroll down to **Artifacts** to download the `playwright-report`

### To Re-run Tests in GitHub

```bash
# Push a commit to trigger the workflow
git add .
git commit -m "Run tests"
git push origin main

# Or manually trigger from GitHub UI:
# Actions > Playwright Tests > Run workflow > branch selector > Run workflow
```

## Jenkins Setup

### Prerequisites

- Jenkins server (≥2.426) running
- Node.js 18+ installed on Jenkins agents
- Git plugin installed
- HTML Publisher plugin installed (`htmlpublisher`)

### Installation Steps

#### 1. Install Required Plugins (Jenkins UI)

Go to **Manage Jenkins** → **Manage Plugins** → **Available**:
- Search and install:
  - **HTML Publisher** (`htmlpublisher`)
  - **JUnit Plugin** (usually pre-installed)
  - **Git Plugin** (usually pre-installed)

#### 2. Create a New Pipeline Job

1. Click **New Item** in Jenkins
2. Enter job name: `Playwright-Automation-Tests`
3. Select **Pipeline**
4. Click **OK**

#### 3. Configure Pipeline

Under **Pipeline** section:
- **Definition**: `Pipeline script from SCM`
- **SCM**: `Git`
- **Repository URL**: `https://github.com/YOUR-USERNAME/PWAutomationAI.git`
- **Branch Specifier**: `*/main` (or `*/master`)
- **Script Path**: `Jenkinsfile` (default)

#### 4. Configure Build Triggers (Optional)

To automatically trigger on GitHub pushes:

- Enable: **GitHub hook trigger for GITScm polling**
  - This requires a **GitHub webhook** (see below)

#### 5. Save and Test

1. Click **Save**
2. Click **Build Now** to test
3. Watch the build logs in real-time
4. After build completes, view the **Playwright Test Report** link

## GitHub Webhook Integration (GitHub → Jenkins)

To automatically trigger Jenkins builds when you push to GitHub:

### On GitHub

1. Go to your repository settings: `https://github.com/YOUR-USERNAME/PWAutomationAI/settings`
2. Click **Webhooks** → **Add webhook**
3. Fill in:
   - **Payload URL**: `http://YOUR-JENKINS-URL/github-webhook/`
   - **Content type**: `application/json`
   - **Events**: `Just the push event`
   - **Active**: ✅ Checked
4. Click **Add webhook**

### On Jenkins

1. Go to job configuration: **Configure**
2. Under **Build Triggers**, enable **GitHub hook trigger for GITScm polling**
3. Save

Now every GitHub push will automatically trigger a Jenkins build!

## View Test Reports

### GitHub Actions

1. Go to **Actions** tab in your repo
2. Select the latest workflow run
3. Download `playwright-report` artifact
4. Extract and open `index.html` in a browser

### Jenkins

1. Go to your Jenkins job
2. Click the build number (e.g., `#1`, `#2`)
3. Look for **Playwright Test Report** link in the left sidebar
4. Click to view detailed HTML report

## Troubleshooting

### GitHub Actions

| Issue | Fix |
|-------|-----|
| Tests fail on Ubuntu but pass locally | Ensure `npx playwright install --with-deps` runs (system deps needed) |
| Auth state (state.json) missing | Ensure `auth.json` is generated by @smoke test or committed to repo |
| Artifact not uploading | Check `.gitignore` – don't ignore `playwright-report/` |

### Jenkins

| Issue | Fix |
|-------|-----|
| Pipeline fails to parse Jenkinsfile | Verify Jenkinsfile syntax: `sh 'npm ci'` (not `npm install`) |
| Test report not showing | Install **HTML Publisher** plugin (`Manage Jenkins` > `Manage Plugins`) |
| Node.js not found | Configure Jenkins node/agent with Node.js installed, or use Docker agent |
| GitHub webhook not triggering builds | Verify webhook URL is reachable & Jenkins has GitHub plugin installed |

## Commands for Local Testing

Before pushing, test locally:

```bash
# Install dependencies
npm ci

# Run all tests
npx playwright test

# Run @smoke tests only
npx playwright test --grep "@smoke"

# Run with debug mode
npx playwright test --debug

# View test report
npx playwright show-report
```

## Environment Variables (Optional)

To pass secrets (API keys, auth tokens, etc.) to tests:

### GitHub Actions

Add to `.github/workflows/playwright.yml`:

```yaml
env:
  API_KEY: ${{ secrets.API_KEY }}
  BASE_URL: ${{ secrets.BASE_URL }}
```

Add secrets in **Settings** → **Secrets and variables** → **Actions** → **New repository secret**

### Jenkins

1. Go to job **Configure**
2. Enable **Use secret text(s) or file(s)**
3. Add credentials and bind to environment variable names
4. Reference in tests via `process.env.API_KEY`
